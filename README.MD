# Security Engineering

This repository holds the **Security Engineering** meta-prompt: threat modeling, secure coding patterns, vulnerability management, and operational security controls for AI-based software development.

## Structure
- `AGENT.md` - Agent constraints and security enforcement rules
- `SPEC.md` - Secure architecture and controls specification
- `BUILD_LOG.md` - Security review and verification log

## Usage
1. Start with this meta-prompt when security is a project requirement.
2. Use it to define threat models, testing strategies, and hardening requirements.
3. Reference it from other meta-prompts as needed.
